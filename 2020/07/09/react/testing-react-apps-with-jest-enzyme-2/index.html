The roots option in our jest config specifies the list of directories that Jest will search for test files, in our case, it will search in the src directory.

The moduleFileExtensions option contains an array of file extensions that our test modules use. In our case, we are only using js and jsx extensions.

The transform object has transformers, i.e modules that transform source files. In our case, we are using babel-jest to transform JavaScript code into a backwards-compatible version of JavaScript, and jest-transform-stub to transform our assets.

The snapshotSerializers option allows us to specify an array of paths to snapshot serializer modules. In our case, we pass the path to the enzyme-to-json serializer.

The setupFilesAfterEnv option allows us to specify a list of paths to modules that contain code to set up our testing framework. In our case, we pass the path to the file that sets up our enzyme configuration.

By default, Jest expects standard JavaScript. The code we write in our React components, JSX, would not be recognized by our tests. We need add the appropriate Babel configuration, which would compile the JSX code into standard JavaScript when Jest runs its test.

Testing the Application Using Jest and Enzyme

We want our test to confirm the following app behaviors:

  1. Our Album component renders photos passed to it correctly.
  2. That each photo in the album receives its required props.
  3. Photos are fetched by default when the input component mounts.
  4. Changing the ID in the input box, and clicking on the search button triggers an HTTP call to fetch photos.

In the above code, we first import React, our Album and Photo components as dependencies. We also import our usePhotoState() custom hook from our context and the shallow() method from the Enzyme library.

Since we do not want to make an actual HTTP call to our API, we mock our context and stub our usePhotoState() custom hook in the beforeEach() block to return an array of photos. In the afterEach() block, we reset all our mocks to their initial state.

Our first test case performs a snapshot test, to check that our UI does not change unexpectedly and that it renders correctly. To do this we use Enzyme’s shallow rendering API. To use it, we pass our component as an argument of the shallow() method. The shallow() method returns a shallow wrapper which we can then assert to check if it matches the snapshot of our UI.

In our second test case, we check the render tree of our wrapper for a selector called .album and expect it to occur only once. We also check that our tree contains 3 Photo nodes, corresponding to each object of our photo array.

Our first test case, “renders input component”, is a snapshot test. It checks that what’s rendered matches our snapshot, a recording of what components are displayed on the screen by Jest.

The second test case, “renders search, input and button components”, similarly checks that the render tree of the wrapper has nodes with selectors named .search, input and button respectively.

Our third test case, “fetches photos on component mount”, checks that the useEffect() hook is called with a function as the first argument and an empty array of dependencies as the second. We use the expect.Any() asymmetric matcher, since deep-equality matchers like .toEqual() compare functions according to referential identity which means they will throw an inequality error and fail our tests. To learn more about this, you can see this issue on the Jest repo here.

Conclusion

In this article, we were able to create a React application that makes an HTTP call to a REST API to display photos on the browser. We were able to see how we can set up a React app using create-react-app and npx and configured axios as our HTTP client.

After creating the application, we configured Jest and Enzyme so we can write unit for the Album and Input React components. With Enzyme and Jest, we were able to test how the components behaved and what they displayed.

To read more concerning how you can leverage Jest and Enzyme to test you React application, you can check out the official Jest documentation and official Enzyme documentation respectively.
